Gemfile:


  source "https://rubygems.org"
  ruby RUBY_VERSION

  # Hello! This is where you manage which Jekyll version is used to run.
  # When you want to use a different version, change it below, save the
  # file and run `bundle install`. Run Jekyll with `bundle exec`, like so:
  #
  #     bundle exec jekyll serve
  #
  # This will help ensure the proper Jekyll version is running.
  # Happy Jekylling!
  gem "jekyll", "3.3.1"

  # This is the default theme for new Jekyll sites. You may change this to anything you like.
  gem "minima", "~> 2.0"

  # If you want to use GitHub Pages, remove the "gem "jekyll"" above and
  # uncomment the line below. To upgrade, run `bundle update github-pages`.
  # gem "github-pages", group: :jekyll_plugins

  # If you have any plugins, put them here!
  group :jekyll_plugins do
     gem "jekyll-feed", "~> 0.6"
  end

Git, Github, and Github Pages

Git is version control software.

There are three commands you need to know right now:


  $ git add  # stages code changes being tracked

  $ git commit -m  # commits the changes to a "save point"

  $ git push  # sends the updated code changes with the save history to a remote repo

Sign up for a Github.com account

https://github.com

Follow the instructions to setup your account: https://help.github.com/articles/set-up-git/

Ask for help if you run into questions!

Create a repository:

username.github.io

In your terminal:

  1. 
  $ git init
            
  2. 
  $ git add .
            
  3. 
  $ git commit -m "Adding my first Jekyll site"
            
  4. 
  $ git remote add origin git@github.com:username/username.github.io.git
            
  5. 
  $ git push origin master
